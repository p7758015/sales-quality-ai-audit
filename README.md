# AI-аудит звонков отдела продаж

Веб-сервис для автоматического анализа качества звонков менеджеров по продажам.  
Обрабатывает текст или аудиозапись разговора, оценивает качество по шкале 0-100% и формирует структурированный отчёт с рекомендациями.

Сервис легко интегрируется с любой CRM или системой телефонии через API и вебхуки.

---

## Problem

Руководителям отдела продаж сложно быстро и объективно оценивать качество звонков менеджеров.  
Ручной аудит занимает много времени, субъективен и плохо масштабируется.

---

## Solution

AI-сервис автоматически анализирует звонки (аудио или текст) и:

- оценивает качество разговора (0-100%)
- выявляет сильные и слабые стороны
- формирует структурированные рекомендации
- помогает стандартизировать контроль качества продаж

Подходит для:

- обучения менеджеров  
- контроля качества звонков  
- автоматизации аудита коммуникаций  

---

## How it works

1. Пользователь загружает аудио или текст звонка  
2. Аудио (при необходимости) транскрибируется через Whisper  
3. Текст передаётся в LLM (GPT)  
4. Модель анализирует диалог по заданным критериям  
5. Формируется:
   - итоговый балл  
   - краткий разбор  
   - рекомендации  

---

## Architecture
```text
User → Web Interface / API → FastAPI backend  
                         ↓  
                   Whisper (speech-to-text)  
                         ↓  
                       GPT (analysis)  
                         ↓  
             Scoring + Summary + Recommendations  
```
---

## Возможности

- Загрузка текста диалога или аудиозаписи звонка  
- Оценка качества разговора по шкале 0–100%  
- Краткий разбор и рекомендации на русском языке  
- Три режима анализа:
  - **Manager+** - акцент на работе менеджера  
  - **Client+** - акцент на реакциях клиента  
  - **Диалог** - общее качество разговора  
- REST API для интеграции:
  - анализ текста  
  - анализ аудио  
  - анализ по ссылке на запись

## Интеграция с CRM и телефонией

Сервис легко интегрируется с любой CRM или системой телефонии через API или вебхуки.

Пример сценария:

1. После завершения звонка CRM отправляет webhook с ссылкой на запись  
2. Сервис автоматически анализирует звонок  
3. Результат возвращается или сохраняется в CRM  

Пример запроса:

```json
POST /webhook/new_call_recording
{
  "call_id": "123",
  "manager_id": "m-42",
  "recording_url": "https://storage.example.com/calls/123.mp3"
}
```
Результат:

- оценка звонка  
- разбор  
- рекомендации  

Может отображаться прямо в карточке сделки или клиента.

## API

### Анализ текста

POST `/analyze_call`

```json
{
  "dialog_text": "Клиент: Привет! Менеджер: Добрый день!",
  "content_mode": "Manager+"
}
```

---

### Анализ аудио

POST `/analyze_audio_call` (multipart/form-data)

---

### Анализ по ссылке

POST `/analyze_audio_by_url`

```json
{
  "file_url": "https://example.com/call_123.mp3",
  "content_mode": "Диалог"
}
```

## Limitations

* качество анализа зависит от точности транскрипции
* LLM может допускать неточности (hallucinations)
* результаты требуют проверки в критичных бизнес-процессах
